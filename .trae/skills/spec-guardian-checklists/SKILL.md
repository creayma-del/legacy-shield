# Spec Guardian — Checklists

本子 Skill 定义 Spec 评审预审清单及归档前最终检查清单。

## Spec 评审预审清单

在调用 Spec 评审专家团之前，先按以下清单自检：

### 完整性

- [ ] 目标、范围、交付物清单是否清晰。
- [ ] 实现步骤是否可执行、无重大遗漏。
- [ ] 测试用例是否覆盖正常路径、异常路径、回归路径。
- [ ] 验收标准是否可逐项核查。

### 一致性

- [ ] Spec 中的技术方案与上游设计文档是否一致。
- [ ] Spec 中的任务拆分与执行计划是否一致。
- [ ] 新增子类型、接口、参数是否在类型定义、日志、分析、API、文档中同步。

### 可落地性

- [ ] 代码片段是否可直接使用或与现有架构兼容。
- [ ] 测试依赖（如夹具、第三方库、环境变量）是否明确。
- [ ] 风险与应对措施是否具体。

### 边界与回归

- [ ] 是否明确说明不在范围内的事项。
- [ ] 是否评估了对 v1.0 或已归档能力的回归风险。
- [ ] 是否包含兼容性设计。

### 文档与状态

- [ ] 文件头是否包含版本、对应文档、状态、评审记录。
- [ ] 文档内部链接是否有效。
- [ ] 状态是否为「评审中」或「待最终评审」。

## 归档前最终检查清单

在将 Spec 状态改为「已完成，已归档」之前，必须确认：

- [ ] 对应代码已开发完成并合并。
- [ ] 所有单元测试、集成测试、端到端测试已通过。
- [ ] 测试验收专家已出具验收通过结论。
- [ ] 用户文档（README、usage、api 等）已同步更新。
- [ ] 验收报告已创建并包含测试结果、需求覆盖、遗留问题。
- [ ] 类型检查、构建、lint 等工程检查已通过（如项目有此类要求）。
- [ ] 所有 P0/P1 缺陷已闭环；P2 优化项已明确是否遗留。
- [ ] 所有任务 Spec 的状态已先更新为「已完成，已归档」。
- [ ] 阶段 Spec 的状态后更新为「已完成，已归档」（阶段 Spec 归档代表该版本全部任务已验收完成）。
- [ ] 设计文档、执行计划的状态已同步更新为「已完成，已归档」。
