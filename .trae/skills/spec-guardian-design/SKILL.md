# Spec Guardian — Design Stage

本子 Skill 覆盖设计阶段：设计文档、执行计划、强制评审门禁。

## 阶段范围

设计文档（评审通过） → 执行计划（评审通过）

## 强制评审门禁

| 文档 | 评审方式 | 阻塞的下一阶段 |
|---|---|---|
| 设计文档 | 调用评审专家正式评审 | 执行计划编写 |
| 执行计划 | 调用评审专家正式评审 | 阶段 Spec 编写 |

**禁止行为**：
- 不得从「草稿」直接标记为「已通过」。
- 不得在未调用评审专家的情况下自行判定通过。
- 不得在评审未通过时继续编写下一阶段文档。

## 规则（if-then）

```
IF 设计文档不存在 → 检查需求阶段前置条件 → 通过则允许创建（草稿）
IF 设计文档状态 == "草稿" → 阻止编写执行计划，提示提交评审
IF 设计文档状态 == "评审中" → 阻止编写执行计划，提示等待评审
IF 设计文档状态 == "已通过" AND 执行计划不存在 → 允许创建执行计划（草稿）
IF 执行计划状态 == "草稿" → 阻止编写阶段 Spec，提示提交评审
IF 执行计划状态 == "评审中" → 阻止编写阶段 Spec，提示等待评审
IF 执行计划状态 == "已通过" → 允许进入 Spec 阶段
```

## 设计文档规则

- 必须在需求分解文档获批准后编写。
- 包含：总体架构、模块详细设计、数据流与状态变更、接口设计、依赖与集成、非功能性设计、风险与回滚方案。
- 文件命名：`docs/specs/design-v{x}.{y}.md`。
- 初始状态：草稿。

## 执行计划规则

- 必须在设计文档评审通过后编写。
- 包含：任务清单、里程碑、资源与角色、风险与应对、任务 Spec 拆解说明。
- 文件命名：`docs/specs/execution-plan-v{x}.{y}.md`。
- 初始状态：草稿。

## 阶段转换检查点（设计 → Spec）

| 检查项 | 通过条件 | 失败动作 |
|---|---|---|
| 设计文档 | 状态为「已通过」（经评审专家评审） | 阻止，提示提交评审 |
| 执行计划 | 状态为「已通过」（经评审专家评审） | 阻止，提示提交评审 |
