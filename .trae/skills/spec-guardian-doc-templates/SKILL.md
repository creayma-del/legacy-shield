# Spec Guardian — Document Templates

本子 Skill 存放所有文档内容模板。

## 需求对齐会议纪要模板

```markdown
# 需求对齐会议纪要

> 版本：v{x}.{y}
> 会议时间：YYYY-MM-DD HH:mm
> 记录人：{SOLO Coder}

## 参与人员
- {用户 / 产品负责人}：{姓名}
- {SOLO Coder}：{姓名}

## 需求条目确认
| 需求编号 | 需求描述 | 确认结论 | 备注 |
|---|---|---|---|
| REQ-{x}-{n} | {描述} | 确认 / 需修订 | {备注} |

## 验收标准确认
- {标准 1}
- {标准 2}

## 签字确认
- 用户 / 产品负责人：{签名} {日期}
- SOLO Coder：{签名} {日期}
```

> 在 IDE 场景下，「签字」指用户明确回复「确认」「同意」。

## PATCH 影响评估模板

```markdown
# PATCH 影响评估
> 对应 Spec：{路径}
> 提出日期：YYYY-MM-DD

## 变更描述
## 影响范围
| 检查项 | 是否影响 | 说明 |
## 是否符合 PATCH 判定条件
- [ ] 不新增 API / CLI 参数 / 配置项
- [ ] 不修改现有接口签名
- [ ] 不引入新的外部依赖
- [ ] 不影响已有测试断言
结论：符合 / 不符合
```
