# Spec Guardian — Document Templates

本子 Skill 存放需求分解文档、设计文档、执行计划的模板。

## 需求分解文档模板

```markdown
# {项目名} v{x}.{y} 需求分解文档

> 版本：v{x}.{y}
> 对应需求文档：requirements-v{x}.{y}.md
> 对应会议纪要：meetings/requirements-alignment-v{x}.{y}-YYYYMMDD.md
> 状态：草稿

## 1. 功能分解

| 功能点编号 | 名称 | 描述 | 对应需求 | 验收标准 | 优先级 |
|---|---|---|---|---|---|
| F-{n} | {名称} | {描述} | REQ-{x}-{n} | {验收标准} | P0/P1/P2 |

## 2. 任务依赖关系

| 任务 | 依赖 | 可并行任务 |
|---|---|---|
| T1 | 无 | T2 |
| T2 | T1 | T3 |

## 3. 资源分配估算

| 任务 | 工作量 | 角色 | 外部资源 |
|---|---|---|---|
| T1 | 中 | 开发专家 | 无 |

## 4. 时间线里程碑

| 里程碑 | 预计日期 |
|---|---|
| 需求对齐完成 | YYYY-MM-DD |
| 需求分解文档批准 | YYYY-MM-DD |
| 设计文档评审通过 | YYYY-MM-DD |
| 阶段 Spec 评审通过 | YYYY-MM-DD |
| 代码开发完成 | YYYY-MM-DD |
| 测试验收通过 | YYYY-MM-DD |
| 归档关闭 | YYYY-MM-DD |

## 5. 风险与假设

| 风险 / 假设 | 说明 | 应对措施 |
|---|---|---|
| {风险 1} | {说明} | {措施} |
```

```markdown
## 2. 模块详细设计

- {对外接口}

## 4. 接口设计

## 5. 依赖与集成

- {外部依赖}

- {兼容性考虑}

## 7. 风险与回滚方案
```

## 执行计划模板

```markdown
# {项目名} v{x}.{y} 执行计划

## 2. 里程碑

| 里程碑 | 预计日期 | 交付物 |
|---|---|---|
| {里程碑 1} | YYYY-MM-DD | {交付物} |

## 3. 资源与角色

- {角色 1}：{职责}
- {角色 2}：{职责}

## 4. 风险与应对

| 风险 | 影响 | 应对措施 |
|---|---|---|
| {风险 1} | {影响} | {措施} |

## 5. 任务 Spec 拆解说明

- 每个任务在计划评审通过后，必须拆解为独立的任务 Spec。
- 任务 Spec 文件命名：`docs/specs/phases/phase-v{x}.{y}-t{n}-spec.md`。
- 所有任务 Spec 评审通过后，方可进入代码开发阶段。
```
