# Spec Guardian — Reply Templates

本子 Skill 存放标准回复模板及验收报告模板。

## 标准回复模板

### 模板 A：前置条件不满足，阻止进入下一阶段

```
当前处于【{当前阶段}】，若要进入【{目标阶段}】，必须先完成：

1. {缺失条件 1}
2. {缺失条件 2}
3. {缺失条件 3}

请补充以上环节后，我再继续协助。
```

### 模板 B：阶段 Spec 与所有任务 Spec 已通过，允许进入代码开发

```
阶段 Spec {阶段 Spec 文件路径} 状态为「已通过」。
所有任务 Spec 状态均为「已通过」。
可以进入代码开发阶段。

开发约束：
- 严格遵循 {设计文档路径} 与对应任务 Spec。
- 如需偏离 Spec，必须先升级阶段 Spec 或相关任务 Spec 并重新评审。
- 完成后需调用测试验收专家进行验收。
```

### 模板 C：已归档文档不可直接修改

```
{文件路径} 状态为「已完成，已归档（冻结，不再修改）」，不可直接修改。

请选择以下路径之一：
1. 若仅为错别字/示例修正 → 走 PATCH 流程。
2. 若涉及能力变更或新增 → 通过新版本 Spec（如 v{x+1}.{y}）重新发起流程。
```

### 模板 D：PATCH 级别判定建议

```
该改动属于 PATCH 级别（{原因}），可走 PATCH 流程简化处理。

简化项：
- {可简化项 1}
- {可简化项 2}

仍需完成：
- 补丁任务 Spec 评审通过。
- 测试验收通过。
- 如涉及设计变更，同步更新设计文档。
```

### 模板 E：评审不通过，需修订

```
{文档路径} 评审不通过，存在以下需修订项：

阻塞项（P0）：
- {问题 1}
- {问题 2}

重要问题（P1）：
- {问题 3}

优化项（P2）：
- {问题 4}

请按以上意见修订后，重新提交评审。
```
