# Spec Guardian — Meeting Templates

本子 Skill 存放需求对齐会议纪要模板、PATCH 影响评估模板。

## 需求对齐会议纪要模板

```markdown
# 需求对齐会议纪要

> 版本：v{x}.{y}
> 会议时间：YYYY-MM-DD HH:mm
> 会议地点：{线下地点 / 线上链接}
> 记录人：{SOLO Coder}

## 参与人员

- {用户 / 产品负责人}：{姓名}
- {SOLO Coder}：{姓名}
- {需求验收专家（可选）}：{姓名}

## 需求条目确认

| 需求编号 | 需求描述 | 确认结论 | 备注 |
|---|---|---|---|
| REQ-{x}-{n} | {描述} | 确认 / 需修订 | {备注} |

## 验收标准确认

- {验收标准 1}
- {验收标准 2}

## 成功指标确认

- {指标 1}：{目标值}
- {指标 2}：{目标值}

## 风险与约束

| 风险 | 影响 | 应对措施 |
|---|---|---|
| {风险 1} | {影响} | {措施} |

## 遗留问题

| 问题 | 责任人 | 解决时限 |
|---|---|---|
| {问题 1} | {责任人} | {时限} |

## 签字确认

- 用户 / 产品负责人：{签名} {日期}
- SOLO Coder：{签名} {日期}
```

> 在 IDE / 聊天场景下，「签字」指用户明确回复「确认」「同意」等文字确认。

## PATCH 影响评估模板

```markdown
# PATCH 影响评估

> 对应 Spec：{Spec 文件路径}
> 提出日期：YYYY-MM-DD
> 提出人：{姓名 / 角色}

## 变更描述

- {变更内容简述}

## 影响范围

| 检查项 | 是否影响 | 说明 |
|---|---|---|
| 公共 API / CLI 接口 | 是 / 否 | {说明} |
| 配置项 | 是 / 否 | {说明} |
| 外部依赖 | 是 / 否 | {说明} |
| 已有测试断言 | 是 / 否 | {说明} |
| 用户文档 | 是 / 否 | {说明} |

## 是否符合 PATCH 判定条件

- [ ] 不新增 API / CLI 参数 / 配置项
- [ ] 不修改现有接口签名或返回结构
- [ ] 不引入新的外部依赖
- [ ] 不影响已有测试的断言目标

结论：符合 / 不符合 PATCH 条件。

## 如不符合，建议版本级别

- {MAJOR / MINOR / 需进一步澄清}

## 评估人

- {姓名}：{日期}
```
